Relatório Técnico — Agente LLM com Tool de Previsão do Tempo

Candidato: Gabriel Willian  |  Data: Abril de 2026  |  Repositório: github.com/GabArtista/teste-climatempo

1. Solução

Sistema implementa um agente conversacional que integra LLM local (Ollama — qwen2.5:1.5b) com a API pública Open-Meteo via function calling no formato compatível com o cliente OpenAI.

Arquitetura Híbrida

A decisão de acionar a tool opera em duas camadas:

  • Camada determinística — _is_weather_query() detecta intenção por keywords; _extract_city() localiza a capital por n-grams + word-boundary regex contra capitals.json. Garante Recall = 1.0 para as 27 capitais, independente do modelo.
  • Camada LLM — O modelo recebe a WEATHER_TOOL (formato OpenAI) e é usado exclusivamente para formatar a resposta final com dados reais da API.
Usuário → POST /api/v1/agent/chat
              │
              ▼
         _is_weather_query()  ── Não ──→  LLM responde livremente
              │ Sim
              ▼
         _extract_city()  ── Não encontrou ──→  LLM pede a cidade
              │ Capital encontrada
              ▼
         WeatherService → GET api.open-meteo.com/v1/forecast
              │
              ▼
         LLM formata resposta com dados reais → Usuário

Essa separação resolve a limitação de modelos pequenos (1.5B) em português: recall baixo ao decidir por tool calling de forma autônoma.

Decisão: phi:latest (inicialmente disponível) foi descartado por não suportar function calling — requisito central do desafio. qwen2.5:1.5b foi escolhido por ser o menor modelo com suporte nativo, rodando dentro das restrições (Intel i5-7200U, 8 GB RAM, sem GPU).

Variáveis Open-Meteo coletadas: temperature_2m_max (°C), temperature_2m_min (°C), precipitation_sum (mm) — conforme especificado no desafio.

3. Validação Quantitativa — Function Calling

Metodologia: amostragem de 18 prompts com classificação binária (8 positivos + 10 negativos). O modelo é tratado como classificador que decide quando acionar a tool.

Recall = 1.0 — A tool foi acionada em 100% dos prompts positivos (zero falsos negativos). O único FP do modelo foi o prompt "Qual é a previsão do tempo?" — intenção de clima corretamente identificada, mas sem cidade. No sistema completo, esse caso é interceptado pela camada determinística: _extract_city() retorna None → agente solicita a cidade ao usuário. Precision efetiva do sistema = 1.0.

Anomalia nos dados detectada e tratada: O arquivo capitals.json continha "Campo Grande - Rio Grande do Norte" (incorreto — capital do RN é Natal). O CapitalsRepository detecta, loga e remove a entrada automaticamente no startup. Endpoint GET /api/v1/weather/data-quality expõe anomalias detectadas.
